# A+D 空间界面升级提示词

> 状态：已确认，可作为设计、实现和验收依据  
> 方向：A `Liquid Archive` 为主，结合 D `Spatial Canvas`  
> 适用范围：登录注册、首页、查 BUG、个人页、文档阅读及现有贡献审核页面

## 1. 总提示词

为“电赛白皮书”设计并实现一套可以真实上线的响应式 Web 产品界面。它是面向电子设计竞赛 Debug 的公开协作知识库，不是营销官网，也不是传统后台管理系统。

视觉以温润的白色空间为画布，正文和技术数据保持清晰、稳定、可长时间阅读；液态玻璃只用于导航、搜索、评论、编辑和状态检查器等真正悬浮的交互层。空间中可以出现低饱和的青色、蓝色和淡紫色折射，但不能覆盖正文、削弱对比度或变成廉价炫光。

取消固定横向菜单栏和常驻管理侧栏。使用左下角径向导航、右上角可扩展搜索球体、页面边缘进度轨道以及从当前内容位置生长的上下文工具。页面之间通过共享元素和轻微空间位移建立连续感，让产品像一个统一工作空间，而不是多个模板页面的拼接。

所有视觉效果必须服务于定位、反馈或状态变化。优先使用原生 CSS、React 状态和少量浏览器 API，不引入仅为动效存在的大型依赖。代码保持短、直观、可维护；复用真正稳定的空间导航、玻璃表面和状态组件，不建立无业务价值的抽象层。

## 2. 设计命题

### 2.1 视觉命题

一张温润、明亮、具有轻微光学折射的技术纸面，少量清澈玻璃控件悬浮其上；能量来自搜索轨道、波形和内容节点的流动，而不是卡片、阴影或装饰图标。

### 2.2 内容命题

- 首页：只有一个核心任务——描述故障并开始搜索。
- 查 BUG：只有一个核心任务——搜索并逐条浏览排查文档。
- 文档：只有一个核心任务——阅读、测量、验证，并在原文位置协作。
- 个人页：只有一个核心任务——查看自己的贡献身份与最近修改。
- 登录注册：只有一个核心任务——进入知识库并建立可追溯身份。
- 提交与审核：保留现有功能，以工作状态、Diff 和明确操作为中心。

### 2.3 交互命题

1. 搜索球体在首页展开为主搜索框，在查 BUG 页缩小为查询工具，在其他页面保持为快捷入口；形态变化连续，而不是突然替换组件。
2. 左下径向导航默认只显示当前入口。悬停、键盘聚焦或点击后，其他入口沿圆弧展开；移动端改为底部紧凑工具岛。
3. 评论、目录、版本和编辑工具从相关内容锚点附近展开。文档选区产生评论入口，进度轨道在评论锚点处扩张，避免固定工具栏占用阅读空间。

## 3. 全局视觉系统

### 3.1 色彩

- 背景：温润白与浅银，不使用纯白大面积刺眼背景。
- 正文：接近黑色的冷灰，确保长文阅读对比度。
- 主操作：深青色，仅用于搜索、当前导航和主要提交操作。
- 验证状态：信号绿，仅表示已验证、通过或成功。
- 光学折射：低饱和青、蓝、淡紫，只出现在画布边缘和玻璃高光。
- 警告与错误：使用语义明确的琥珀和红色，同时提供图标或文字。

### 3.2 字体

- 全站最多两套字体。
- 界面、正文和技术数据以现代中文无衬线字体为主。
- 大标题可使用同一字体的较重字重，不再依赖古典宋体制造知识感。
- 数字、版本和测量值可使用系统等宽字体，但只作为局部功能字体。

### 3.3 材质

- `canvas`：无边框的连续内容空间。
- `glass`：半透明背景、细描边、背景模糊、顶部高光和克制阴影。
- `glass-strong`：仅用于登录面板、评论检查器和弹出的命令框。
- `paper`：正文区域保持接近不透明，不在长文下方使用持续模糊。
- 页面不得堆叠两层以上玻璃容器。

### 3.4 形状

- 搜索球体、径向按钮和评论锚点使用圆形。
- 搜索框、检查器和身份面板可以使用大圆角胶囊或柔和矩形。
- 文档列表、正文区和时间线不使用卡片外框。
- 普通按钮不全部做成胶囊；文本操作保持轻量。

## 4. 公共空间导航

### 4.1 品牌

- “电赛白皮书”固定在画布左上安全区，但不占据整行。
- 品牌只包含名称和一个简单的线路/书页符号。
- 进入页面时品牌轻微淡入，不参与持续动画。

### 4.2 径向导航

- 桌面端位于左下安全区，中心按钮表示当前页面。
- 首页、查 BUG、创建/贡献、个人页沿圆弧展开。
- 当前入口使用深青玻璃，其他入口使用清透玻璃。
- 悬停时按钮产生不超过 4px 的磁吸位移，标签从按钮外侧淡入。
- 键盘聚焦与点击必须提供与悬停等价的展开能力。
- 移动端不使用完整圆环，收为底部四项工具岛。

### 4.3 搜索球体

- 除首页外位于右上安全区。
- 点击后沿水平方向扩展为命令框，自动聚焦输入框。
- `Escape` 收起；提交后前往查 BUG 页并保留查询词。
- 首页直接展示展开状态，但保留右侧发光核心作为提交按钮。

## 5. 页面提示词

### 5.1 登录 / 注册

设计一个单一的身份玻璃面板，通过共享布局在“登录”和“注册”之间变形，不放置两个并排表单。

- 左上显示品牌。
- 背景使用一条抽象、低对比的信号波形和轨道粒子。
- 面板只包含模式切换、用户名、密码、错误信息和主操作。
- 登录按钮文案为“进入知识库”；注册按钮文案为“创建账号”。
- 模式切换时面板高度和字段平滑变化，输入内容不应意外丢失。
- 登录成功后返回来源页面或首页。
- 移动端面板占据大部分宽度，背景波形降级为静态。

### 5.2 首页

设计一个全画布搜索首页，不放文章列表、功能卡片、营销文案或数据看板。

- 搜索框是第一视觉焦点，位于画布中部。
- 四个常见故障短语沿椭圆轨道缓慢错位运动：“无法上电”“输出掉压”“串口乱码”“电机振荡”。
- 鼠标靠近短语时，短语暂停并展开一行解释；点击后将短语填入搜索框。
- 背景信号线向搜索框汇聚，指针移动只产生很小的折射偏移。
- 右下可以显示一条安静的社区状态，例如当前知识条目数量，不承载关键业务。
- 动画在页面首次进入后完成一次主要编排，持续动画速度必须很慢。
- `prefers-reduced-motion` 下轨道和折射停止，只保留静态布局。

### 5.3 查 BUG

设计一个可快速扫描的线性文档索引。

- 页面标题为“查 BUG”，旁边显示真实结果数量。
- 搜索框位于标题右侧或上方，并保持明显但不抢占文档列表。
- 每篇文档是一行，不使用独立卡片；包含序号、标题、摘要、验证状态和进入箭头。
- 搜索命中词使用低饱和高亮，不改变行高。
- 右侧过滤轨道提供“现象”“器件”“最近验证”。当前阶段只实现“现象”查询，其余可以显示为不可用或不渲染，不能伪造功能。
- 列表进入时使用轻微错峰淡入，搜索更新时使用局部过渡，不让整个页面闪烁。
- 移动端隐藏大序号或将其缩小，搜索框置顶，行内容保持可读。

### 5.4 个人页

设计一个贡献者身份和贡献轨迹页面，不做传统仪表盘。

- 展示用户名、角色和一句简短的贡献说明。
- 统计只来自真实数据：贡献总数、已发布、审核中、草稿/驳回。
- 最近贡献沿一条纵向或弧形轨迹排列；选择某条记录后，从节点附近展开详情。
- 记录展示标题、修改说明、状态、更新时间和版本号。
- 审核员可以看到进入审核队列的轻量入口。
- 未登录用户访问时跳转到登录页，并带上返回地址。
- 没有贡献时展示明确空状态和“去查 BUG”入口。

### 5.5 具体文档

设计一个长时间阅读友好的技术文档页。

- 中央正文带保持稳定宽度，不使用外层卡片。
- 标题下展示验证状态、版本、修改者、审核者和日期。
- 左下径向导航在文档页切换为目录、历史、收藏、分享等文档工具；当前只实现已有目录和历史，未实现项不能伪装为可用。
- 右侧细进度轨道根据滚动位置更新，并在评论锚点处形成节点。
- 保留适用范围、安全提示、快速检查、排查步骤、代码、表格和波形。
- 点击“编辑”在原文位置进入现有结构化编辑器，不能跳转独立页面。
- 选中正文文本后，在选区附近显示“评论”上下文按钮；当前若评论后端尚未完成，可以打开本地交互面板并明确提示“评论保存功能将在下一阶段接入”，不能伪造成功。
- 评论检查器默认收起，打开后覆盖右侧空间但不永久压缩正文。
- 移动端隐藏进度轨道，将目录和评论收进底部工具岛。

### 5.6 我的提交与审核

- 继续使用空间画布和公共导航，但信息密度高于首页。
- 提交列表沿时间线展示状态，不堆叠卡片。
- 审核页仍以 Diff 为主，增删内容可辨识并显示作者、审核者和日期。
- 通过与驳回是明确的双操作，驳回原因仍为必填。
- 不为了与首页统一而增加轨道、光球或装饰动画。

## 6. 动效规格

### 6.1 页面进入

- 画布淡入：`opacity 0 → 1`，约 `360ms`。
- 主内容：`translateY(10px) → 0`，约 `520ms`。
- 首页轨道词：间隔 `60–90ms` 错峰出现。

### 6.2 交互

- 径向导航：`transform + opacity`，`220–360ms`，使用快速收尾缓动。
- 搜索球体展开：宽度、透明度和内部元素错峰，`360–480ms`。
- 检查器/评论：从锚点方向位移 `12–20px` 并淡入，`280–420ms`。
- 按钮按下：缩放不低于 `0.97`，`120ms`。
- 列表箭头：悬停平移不超过 `4px`。

### 6.3 限制

- 不使用弹跳、连续旋转、大幅缩放、滚动劫持或快速视差。
- 不让背景滤镜持续重绘整页。
- 不对 `width: auto` 等不可预测属性做复杂动画。
- `prefers-reduced-motion` 下取消轨道运动、指针折射和空间位移。

## 7. 响应式与可访问性

- 必测宽度：`320 / 375 / 414 / 768 / 1280 / 1440px`。
- 所有页面禁止真实横向溢出。
- 所有文本输入、按钮和导航目标至少 `44px`。
- 桌面径向导航在移动端变为底部工具岛，不能缩小整套圆环硬塞。
- 搜索、导航、对话框和评论操作支持键盘。
- 玻璃背景上的文字必须保持 WCAG AA 对比度。
- 焦点环清晰、立即出现，不使用只靠颜色表达的状态。
- 页面地标、标题层级、状态播报和表单错误关联保持语义正确。

## 8. 数据与后端约束

- 前端优先复用现有 `/auth`、`/symptoms`、`/articles`、`/reviews` 接口。
- 个人页只新增一个“我的概览”接口，返回当前用户、各状态贡献计数和最近贡献；由一条简单查询和少量 Python 聚合完成。
- 不新增 Repository、Service、DTO 映射器等当前规模不需要的层。
- 不为尚未实现的评论、收藏和筛选返回假数据。
- 后端响应字段只包含页面实际使用的数据。
- 新接口必须有未登录和正常返回测试。

## 9. 代码约束

- 保持 React 组件职责清晰，但不为每个小图形创建独立组件。
- 稳定复用组件限定为：空间背景、径向导航、搜索球体、玻璃检查器、状态反馈。
- 使用 CSS Modules 和全局 token；组件内不写散落的颜色值。
- 动效优先使用 `transform`、`opacity` 和 CSS 自定义属性。
- 指针折射如需 JavaScript，只在 `requestAnimationFrame` 中更新 CSS 变量，不触发 React 高频重渲染。
- 保留现有 API、编辑、审核和 Diff 业务逻辑，不因为视觉升级重写后端流程。
- 删除被新空间导航完全替代的旧头部和页脚代码，避免两套系统并存。

## 10. 验收标准

1. 五个确认页面与原型保持同一视觉系统。
2. 首屏不再出现传统横向菜单栏或管理后台式侧栏。
3. 首页只有一个主要搜索交互，并且动效可感知但不干扰输入。
4. 查 BUG 页真实搜索、结果数量和文档跳转可用。
5. 个人页数据来自后端真实贡献记录。
6. 文档阅读、原位编辑、版本历史和审核流程无功能回归。
7. 登录注册可用，错误和加载状态明确。
8. 键盘与减少动态效果模式可完成核心操作。
9. 前后端 lint、测试和生产构建通过。
10. 关键页面在六个目标宽度无横向溢出，并通过真实浏览器验收。
